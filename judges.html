What pressing the button does

One push → still + narration + composed sleep widget. In ~15 seconds. End to end.

Atomic pipeline. No queue. No "pending" hand-off. The user presses one button, and a finished playable artifact comes back.

generate_image → speak(pace:"sleep") → compose_sleep_video ~10s — Runway gen4_image ~3s — ElevenLabs slow path instant — CSS Ken Burns cinematic vertical sleep still SSML breath pauses + speed 0.80 stitches image + audio Rachel voice forced synced 30s playback widget
Why no Runway video render in the sleep path? Because the user pressed one button. Adding a 60–90s gen4.5 render breaks the atomic guarantee. We use CSS Ken Burns motion on the still under the narration — a drifting, dreamy sleep visual without any video synthesis. The full Runway video stack is still used elsewhere in the agent (cinematic shorts, character dialogue, atmospheric grading) — just not on the critical-path sleep pipeline that has to finish in one Edge call.

The architecture

Solo dev. Edge runtime. Streamed agent loop. No background workers.

Single-file Edge function (api/agent.js, ~72KB) that runs an Anthropic tool-use loop with full SSE streaming back to the browser. Server-side defenses learned the hard way:

  • Verbatim-script rule — if the user's prompt contains quoted narrator text, the agent passes it verbatim to speak rather than inventing its own monologue.
  • Server-side sleep-pace auto-detect — 24 cadence cues ("close your eyes", "drift", "breathe", "whisper", etc.) auto-promote pace to "sleep" if the model forgets. Belt and suspenders.
  • Atomic pipeline guarantee — sleep videos use generate_image → speak → compose_sleep_video. No animate_image call to risk blowing the Edge budget.
  • One-push hard rule — agent treats the whole pipeline as a single transaction. No queue, no pending hand-off.

Built by one founder (Roger Grubb · Number One Son Software Development LLC) using Claude as a co-CTO. Tech stack: Vercel Edge + Claude API tool-use + Runway API + ElevenLabs API. No backend servers, no databases — the agent IS the product.
